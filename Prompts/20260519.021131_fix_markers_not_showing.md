
# Implementation Plan - Fixing Marker Visibility and Camera Centering

This plan addresses the issue where the markers were not showing up and the map opened at a random or default location. 

## Cause Analysis
1. **Camera Centering Exception**: 
   The initial centering code used `CameraUpdateFactory.newLatLngBounds(bounds, 150)`. In Mapbox/MapLibre, calling `newLatLngBounds` before the `MapView` has undergone a full layout pass (where its width and height are measured and non-zero) throws an `IllegalArgumentException: Cannot zoom to bounds before map has a size`. 
   If this exception is thrown inside `postDelayed`, it halts execution completely. Thus, the camera never centers on the target coordinates (Cochin, India) and stays at the default startup coordinate (usually `0, 0` off the coast of Africa), and the movement loop is never started.
2. **Marker Size & Density**:
   The programmatic marker bitmap size was set to `64x64` pixels. On high-density screens (XXHDPI/XXXHDPI), a 64px image scales down to only ~16-20dp, making the markers virtually invisible specks on the map.

---

## Proposed Fixes

1. **Robust Camera Centering**:
   Immediately center the camera upon style loading using `CameraUpdateFactory.newLatLngZoom(midpoint, 14.5)`. This is a synchronous operation that does **not** depend on the map's layout size and will **never** throw an exception.
2. **Enlarge Markers**:
   Increase the programmatically generated marker size to `128x128` pixels (highly visible on all screen densities).
3. **Enhance Marker Graphics**:
   Redesign the marker bitmap to include an outer drop shadow, a white ring, the main colored ring, and a bright white central core for a highly visible and professional GPS beacon appearance.
4. **Remove Unstable Camera Bound Updates**:
   Replace the layout-dependent `newLatLngBounds` call with direct, stable camera updating to completely eliminate the exception risk.

---

## Detailed Modifications

### 1. Update `MainActivity.java` Startup Sequence
We will modify the style loading completion callback in `MainActivity.java` to:
- Instantly position the camera at the computed midpoint using `moveCamera(newLatLngZoom(midpoint, 14.5))`.
- Setup the enlarged, ultra-visible `128px` Red and Green markers.
- Instantly start the movement loop.

### 2. Update `createMarkerBitmap` Design
```java
private Bitmap createMarkerBitmap(int color) {
    int size = 128;
    Bitmap bitmap = Bitmap.createBitmap(size, size, Bitmap.Config.ARGB_8888);
    Canvas canvas = new Canvas(bitmap);

    Paint paint = new Paint();
    paint.setAntiAlias(true);

    // 1. Shadow (Soft Black)
    paint.setColor(Color.parseColor("#40000000"));
    canvas.drawCircle(size / 2f, size / 2f + 8, size / 2.4f, paint);

    // 2. Outer border (Pure White)
    paint.setColor(Color.WHITE);
    canvas.drawCircle(size / 2f, size / 2f, size / 2.4f, paint);

    // 3. Colored Core
    paint.setColor(color);
    canvas.drawCircle(size / 2f, size / 2f, size / 3.4f, paint);

    // 4. Center Dot (Pure White)
    paint.setColor(Color.WHITE);
    canvas.drawCircle(size / 2f, size / 2f, size / 8f, paint);

    return bitmap;
}
```

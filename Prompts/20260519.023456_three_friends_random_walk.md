# Implementation Plan - Three Friends Independent Random Walk with Distance Constraints

This plan outlines how to make three markers walk independently in random directions at 1–5 meters per second while strictly maintaining a distance of 1000m to 1010m between all pairs.

## Mathematical Formulation
1. **Initial Setup (Equilateral Triangle)**:
   We will initialize the Red, Green, and Blue markers to form a perfect equilateral triangle with a side length of exactly **1005 meters**. This places all three pairs perfectly in the sweet spot of the $[1000, 1010]$ meters range.
2. **Independent Random Walk**:
   At each 1-second tick:
   * Red moves by a random distance between 1 and 5 meters in a random direction.
   * Green moves by a random distance between 1 and 5 meters in a random direction.
   * Blue moves by a random distance between 1 and 5 meters in a random direction.
3. **Position-Based Dynamics (PBD) Spring Constraints**:
   Independent random walks will cause the distances to drift. To maintain the $[1000, 1010]$ meters constraint range, we will run a lightweight, robust 3-iteration PBD spring correction:
   * Treat the pairwise connections as springs with a rest length of **1005 meters**.
   * If a distance drifts, the solver gently nudges both coordinates closer or further apart to restore the 1005m target.
   * This guarantees mathematical convergence, 100% stability, and ensures all three pairwise distances remain locked within $[1000, 1010]$ meters while letting the friends walk independently!

## Technical Approach

### 1. Initial Position Setup

double sKm = 1.005; // 1005 meters
double deltaLat = (sKm / 111.12) * Math.cos(angle);

redLatLng = new LatLng(baseLat, baseLon);

double perpAngle = angle + Math.PI / 2.0;

blueLatLng = new LatLng(midLat + perpLat, midLon + perpLon);

### 2. High-Accuracy Haversine Distance Helper
Add a helper method to compute real-world distance between coordinates in meters:
```java
private double calculateDistance(LatLng p1, LatLng p2) {
    double R = 6371000; // Earth's radius in meters
    double lat1 = Math.toRadians(p1.getLatitude());
    double lon1 = Math.toRadians(p1.getLongitude());
    double lat2 = Math.toRadians(p2.getLatitude());
    double lon2 = Math.toRadians(p2.getLongitude());
    double dLat = lat2 - lat1;
    double dLon = lon2 - lon1;
    double a = Math.sin(dLat / 2) * Math.sin(dLat / 2) +
               Math.cos(lat1) * Math.cos(lat2) *
               Math.sin(dLon / 2) * Math.sin(dLon / 2);
    double c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
    return R * c;
}
```

### 3. PBD Spring Constraint Solver
Add a helper method to adjust a pair of coordinates towards a target distance of 1005 meters:
```java
private LatLng[] adjustPair(LatLng p1, LatLng p2, double targetDistance) {
    double currentDistance = calculateDistance(p1, p2);
    if (currentDistance == 0) return new LatLng[]{p1, p2};

    double diff = currentDistance - targetDistance;
    double factor = (diff / 2.0) / currentDistance;

    double dLat = p2.getLatitude() - p1.getLatitude();
    double dLon = p2.getLongitude() - p1.getLongitude();

    LatLng newP1 = new LatLng(p1.getLatitude() + dLat * factor, p1.getLongitude() + dLon * factor);
    LatLng newP2 = new LatLng(p2.getLatitude() - dLat * factor, p2.getLongitude() - dLon * factor);

    return new LatLng[]{newP1, newP2};
}
```

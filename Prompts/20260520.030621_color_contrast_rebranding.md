# Color & Contrast Optimization Log

## Issue Solved
The user reported that the side drawer menus and the dialog popups had poor color themes and low contrast.

## Changes & Implementation Details

1. **Integrated Modern Premium Drawer Theme**:
   - Replaced the dark `#121214` / `#1E1E24` background with a clean, high-contrast Material light design.
   - Designed a cobalt/midnight royal blue header (`#1A237E`) with highly legible crisp white primary texts and ice blue subtitles (`#C5CAE9`).
   - Tinted all icons inside the drawer dynamic royal blue (`#1A237E`) using `android:tint` to prevent them from blending in or appearing black-on-black on dark setups.
   - Replaced body text colors with high-contrast Material dark charcoal (`#212121`).

2. **Refactored Popup Dialogs & Input Text Contrast**:
   - **Label Colors**: Changed the low-contrast slate `#78909C` labels to use a bold, premium Indigo color (`#1A237E`) with high visibility.
   - **Input Text Colors**: Removed the hardcoded dark text color (`#263238`) from dialog inputs. This allows `EditText` views to automatically inherit the standard, correct theme-defined text colors from system configurations (e.g. perfect white on dark mode, black on light mode), avoiding unreadable black-on-black or white-on-white text issues under all screen conditions.
   - **Table Headers**: Formatted the dual-column rooms list headers as beautiful, bold royal blue (`#1A237E`) with extreme readability.

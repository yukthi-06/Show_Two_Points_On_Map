# User Prompt
The user name set by user in the Settings page, that same user name has to be selected in the "Friends" settings checkbox list by default and un-selectable.

# Walkthrough - Current User Unselectable in Friends List

This walkthrough documents the update to automatically check and disable (make un-selectable) the checkbox of the user configured as the "Current User" in Settings.

## Changes Made

### 1. Friends Checklist Pre-selection & Disabling
- Modified [FriendsActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/01_GPS/Friends_Location_Tracker_APK_GITHUB_LATEST/app/src/main/java/com/vypeensoft/friendtracker/FriendsActivity.java):
  - Added logic to load the configured `"current_user"` from the `"AppConfig"` SharedPreferences.
  - When rendering the dynamic checklist of friends, if a friend's name matches the current user (case-insensitively), its CheckBox is automatically checked (`cb.setChecked(true)`) and disabled (`cb.setEnabled(false)`).
  - This ensures the current user is tracked by default and cannot be unchecked by the user, satisfying the requirement to be un-selectable.

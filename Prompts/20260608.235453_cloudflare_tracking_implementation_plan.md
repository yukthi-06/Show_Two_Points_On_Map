When user click on "Track Them" buton, app goes to the map.
The tracking starts now.  
Follow the steps below to do the tracking...
Get the list of people selected in the previous screen.
Convert them to lower case.
Sort them alphabetically
concat the names , separated by underscore
This becomes the session_id.
Now, repeat the below steps with an interval of "cloudflare polling interval"
    ->get the GPS co-ordinates and extract the latitude and longitude
    ->get the current timestamp in millisecond since epoch
    ->create a json with  sessionid, userName, latitude, longitude, timestamp
    ->send a POST http request to <CLOUDFLARE_URL>/update

# Cloudflare Periodic Tracking Implementation Plan

We will add a periodic HTTP POST task to the background `LocationService`. This task will run at the configured `cloudflarePollingInterval` rate, retrieve the current GPS coordinates, construct the required JSON payload (including the alphabetically ordered, lowercased, and concatenated `sessionid`), and send a POST request to `<CLOUDFLARE_URL>/update`.

#### [MODIFY] [LocationService.java](file:///x:/Projects_X/0_Active/8_Android_APK/01_GPS/Friends_Location_Tracker_APK_GITHUB_LATEST/app/src/main/java/com/vypeensoft/friendtracker/service/LocationService.java)

  - Fetch the set of selected friends from `"friend_tracker_prefs"`.

  - Sort alphabetically.
  - Concatenate names with an underscore.

- Spawn a background Thread to execute an HTTP POST request to `<CLOUDFLARE_URL>/update` with the JSON payload.
- Clean up the repeating runnable on service destruction.

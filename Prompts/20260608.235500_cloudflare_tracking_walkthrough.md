# User Prompt
When user click on "Track Them" buton, app goes to the map.
The tracking starts now.  
Follow the steps below to do the tracking...
Get the list of people selected in the previous screen.
Convert them to lower case.
Sort them alphabetically
concat the names , separated by underscore
This becomes the session_id.
Now, repeat the below steps with an interval of "cloudflare polling interval"
    ->get the GPS co-ordinates and extract the latitude and longitude
    ->get the current timestamp in millisecond since epoch
    ->create a json with  sessionid, userName, latitude, longitude, timestamp
    ->send a POST http request to <CLOUDFLARE_URL>/update

# Walkthrough - Periodic Cloudflare Tracking Updates

We have implemented dynamic GPS tracking reporting to a Cloudflare worker backend inside the background `LocationService`.

## Changes Made

### 1. Dynamic Settings & Location Tracking Loop
- Modified [LocationService.java](file:///x:/Projects_X/0_Active/8_Android_APK/01_GPS/Friends_Location_Tracker_APK_GITHUB_LATEST/app/src/main/java/com/vypeensoft/friendtracker/service/LocationService.java):
  - Added JSON configuration parsing methods `loadCloudflareUrl()` and `loadCloudflareInterval()` which load settings from `/sdcard/Vypeensoft/Friends_Location_Tracker/settings/cloudflare.json`.
  - Stored `lastKnownLocation` inside the `LocationCallback` event handler to track the most recent coordinates.
  - Implemented `startCloudflareLoop()` using a `Handler` loop running every `cloudflarePollingInterval` seconds.
  - Formatted the session ID by getting checked usernames, converting to lowercase, sorting alphabetically, and joining with an underscore.
  - Built the JSON payload using keys: `sessionid`, `userName`, `latitude`, `longitude`, and `timestamp`.
  - Dispatched the updates as an HTTP POST request inside a worker thread using `HttpURLConnection` to `<CLOUDFLARE_URL>/update`.
  - Integrated start and stop hooks into `onStartCommand` and `onDestroy` lifecycle events of the service.

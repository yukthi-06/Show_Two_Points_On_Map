# User Prompt
when user click on Stop Tracking button, invoke end point /delete with sessionid and userName.  It returns the remaining users as response.
Show only current user on map and not  other users

# Walkthrough

## Changes Made

### MainActivity
- Modified `MainActivity.java`'s `startMovementLoop()` coordinates reading block to filter `loadedLocations` such that it only retains the current user. This prevents other participants' markers from appearing on the map.
- Modified `updateFriendMarker` in `MainActivity.java` to check if the incoming username matches the current user. If it does not match, it ignores the update and removes any existing marker for that username.
- Updated the "Stop Tracking" button click listener to retrieve the Cloudflare URL, build the `sessionid` and `userName`, and invoke `/delete` in a background thread.
- Added helper methods `loadCloudflareUrl()`, `sendCloudflareDelete()`, `processCloudflareDeleteResponse()`, and `writeParticipantLocationToSessions()` in `MainActivity.java` to support the `/delete` API request, parse remaining users from the JSON response, and clean up stale files in `/sdcard/Vypeensoft/Friends_Location_Tracker/sessions/`.
